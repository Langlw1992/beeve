Sidebar 组件用于构建应用程序的侧边栏导航区域，支持多种折叠模式。

## 基础用法

<SidebarBasic client:only="solid-js" />

```tsx
import { Sidebar, NavMenu } from '@beeve/ui'

<Sidebar.Provider defaultOpen collapsible="icon">
  <Sidebar>
    <Sidebar.Header>
      <span>Logo</span>
    </Sidebar.Header>
    <Sidebar.Content>
      <NavMenu items={menuItems} />
    </Sidebar.Content>
    <Sidebar.Footer>
      <Sidebar.Trigger />
    </Sidebar.Footer>
  </Sidebar>
  <main class="flex-1">主内容区域</main>
</Sidebar.Provider>
```

## 图标折叠模式

<SidebarCollapsibleIcon client:only="solid-js" />

`collapsible="icon"` 模式下，折叠时仅显示图标：

```tsx
<Sidebar.Provider defaultOpen={false} collapsible="icon">
  <Sidebar>
    {/* ... */}
  </Sidebar>
</Sidebar.Provider>
```

## 抽屉模式

<SidebarOffcanvas client:only="solid-js" />

`collapsible="offcanvas"` 模式下，折叠时完全隐藏：

```tsx
<Sidebar.Provider defaultOpen={false} collapsible="offcanvas">
  <Sidebar>
    {/* ... */}
  </Sidebar>
  <main>
    <Sidebar.Trigger>
      <Button>打开侧边栏</Button>
    </Sidebar.Trigger>
  </main>
</Sidebar.Provider>
```

## 受控模式

<SidebarControlled client:only="solid-js" />

```tsx
import { createSignal } from 'solid-js'
import { useSidebar } from '@beeve/ui'

const [open, setOpen] = createSignal(true)

<Sidebar.Provider open={open()} onOpenChange={setOpen} collapsible="icon">
  <Sidebar>
    {/* ... */}
  </Sidebar>
  <main>
    <Button onClick={() => setOpen(!open())}>
      {open() ? '折叠' : '展开'}
    </Button>
    <p>快捷键: Cmd/Ctrl + B</p>
  </main>
</Sidebar.Provider>
```

## 不可折叠

<SidebarNone client:only="solid-js" />

`collapsible="none"` 模式下，侧边栏始终展开：

```tsx
<Sidebar.Provider collapsible="none">
  <Sidebar>
    {/* ... */}
  </Sidebar>
</Sidebar.Provider>
```

## 组件结构

```tsx
<Sidebar.Provider>      {/* 提供上下文和布局容器 */}
  <Sidebar>             {/* 侧边栏主体 */}
    <Sidebar.Header>    {/* 头部区域（Logo 等） */}
    <Sidebar.Content>   {/* 内容区域（导航等） */}
    <Sidebar.Footer>    {/* 底部区域（触发器等） */}
      <Sidebar.Trigger> {/* 折叠/展开按钮 */}
    </Sidebar.Footer>
  </Sidebar>
  <main>...</main>      {/* 主内容区域 */}
</Sidebar.Provider>
```

## API

### Sidebar.Provider Props

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `open` | `boolean` | - | 是否展开（受控） |
| `defaultOpen` | `boolean` | `true` | 默认是否展开（非受控） |
| `onOpenChange` | `(open: boolean) => void` | - | 展开状态变化回调 |
| `collapsible` | `'offcanvas' \| 'icon' \| 'none'` | `'icon'` | 折叠模式 |

### useSidebar Hook

| 属性 | 类型 | 说明 |
|------|------|------|
| `state` | `Accessor<'expanded' \| 'collapsed'>` | 当前状态 |
| `open` | `Accessor<boolean>` | 是否展开 |
| `setOpen` | `(open: boolean) => void` | 设置展开状态 |
| `toggleSidebar` | `() => void` | 切换展开状态 |
| `collapsible` | `Accessor<CollapsibleMode>` | 折叠模式 |

### Sidebar / Sidebar.Header / Sidebar.Content / Sidebar.Footer Props

这些组件都接受标准的 `div` 属性，包括 `class` 和 `children`。

### Sidebar.Trigger Props

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `class` | `string` | - | 自定义类名 |
| `children` | `JSX.Element` | - | 自定义触发器内容 |

触发器在 `collapsible="none"` 模式下不会渲染。
